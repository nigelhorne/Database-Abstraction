   66: =head1 DESCRIPTION
   67: 
   68: C<Database::Abstraction> is a read-only ORM for Perl that gives a uniform
   69: interface over CSV, PSV, XML, SQLite, and BerkeleyDB files â€” without writing
   70: any SQL.
   71: 
   72: Key features:
   73: 
   74: =over 4
   75: 
   76: =item *
   77: 
   78: B<No SQL required.>  Use plain Perl method calls for simple lookups and
   79: scans; switch storage formats without changing application code.
   80: 
   81: =item *
   82: 
   83: B<Rich query criteria.>  Pass plain values, SQL wildcards, C<undef> (IS NULL),
   84: comparison operators (C<< > >> C<< < >> C<< >= >> C<< <= >> C<!=>), pattern
   85: operators (C<-like>, C<-not_like>), set operators (C<-in>, C<-not_in>,
   86: C<-between>), and logical groupings (C<-or>, C<-and>).
   87: 
   88: =item *
   89: 
   90: B<Automatic joins.>  Add a C<join> parameter to any select method to
   91: combine tables with INNER, LEFT, RIGHT, FULL, or CROSS joins.
   92: 
   93: =item *
   94: 
   95: B<Chained query builder.>  The C<query()> method returns a
   96: L<Database::Abstraction::Query> object for fluent, composable queries:
   97: C<< $db->query->where(â€¦)->order_by(â€¦)->limit(â€¦)->all() >>.
   98: 
   99: =item *
  100: 
  101: B<Schema introspection.>  C<columns()> lists column names; C<schema()>
  102: returns full type/nullability metadata, using native driver introspection
  103: (C<PRAGMA table_info> for SQLite, C<column_info> for others).
  104: 
  105: =item *
  106: 
  107: B<DSN portability.>  Pass a C<dsn> (plus optional C<username>/C<password>)
  108: to connect to any DBI-supported database (SQLite, PostgreSQL, MySQL, â€¦)
  109: instead of pointing at a local file.
  110: 
  111: =item *
  112: 
  113: B<Performance.>  Small files are slurped into a RAM hash for sub-millisecond
  114: lookups.  All DBI statement handles are cached with C<prepare_cached()>.
  115: A CHI-compatible cache layer is also supported.
  116: 
  117: =back

 2492: =head1 KNOWN LIMITATIONS
 2493: 
 2494: =over 4
 2495: 
 2496: =item *
 2497: 
 2498: B<Read-only.>  No INSERT, UPDATE, or DELETE is provided.  C<execute()>
 2499: runs raw read-only SQL.
 2500: 
 2501: =item *
 2502: 
 2503: B<Default CSV separator is C<!>>, not C<,>, for historical reasons.
 2504: Pass C<< sep_char => ',' >> for standard RFC 4180 files.
 2505: 
 2506: =item *
 2507: 
 2508: B<Primary-key column is named C<entry>>, not C<key>, because C<key>
 2509: is a SQL reserved word.  Override with the C<id> parameter.
 2510: 
 2511: =item *
 2512: 
 2513: B<XML slurp is limited.>  Only simple flat XML structures are supported
 2514: in slurp mode.  Multi-key or deeply nested documents will croak.
 2515: Force SQL mode with C<< max_slurp_size => 0 >> if slurp fails.
 2516: 
 2517: =item *
 2518: 
 2519: B<Unique key assumption in slurp mode.>  Duplicate values in the key
 2520: column silently overwrite earlier rows.  Disable slurp with
 2521: C<< max_slurp_size => 0 >> if duplicates are expected.
 2522: 
 2523: =item *
 2524: 
 2525: B<BerkeleyDB does not support joins or the chained query builder.>
 2526: 
 2527: =item *
 2528: 
 2529: B<Column names must be valid SQL identifiers> (letters, digits,
 2530: underscores, and a single dot for C<table.column> join notation).
 2531: Other characters will cause a croak.
 2532: 
 2533: =item *
 2534: 
 2535: B<count() cache is opportunistic.>  Count results are served from cache
 2536: only when a prior C<selectall_arrayref()> or C<count()> call with the
 2537: same criteria has already populated it.
 2538: 
 2539: =back
