   69: =head1 DESCRIPTION
   70: 
   71: C<Database::Abstraction> is a read-only ORM for Perl that gives a uniform
   72: interface over CSV, PSV, XML, SQLite, and BerkeleyDB files â€” without writing
   73: any SQL.
   74: 
   75: Key features:
   76: 
   77: =over 4
   78: 
   79: =item *
   80: 
   81: B<No SQL required.>  Use plain Perl method calls for simple lookups and
   82: scans; switch storage formats without changing application code.
   83: 
   84: =item *
   85: 
   86: B<Rich query criteria.>  Pass plain values, SQL wildcards, C<undef> (IS NULL),
   87: comparison operators (C<< > >> C<< < >> C<< >= >> C<< <= >> C<!=>), pattern
   88: operators (C<-like>, C<-not_like>), set operators (C<-in>, C<-not_in>,
   89: C<-between>), and logical groupings (C<-or>, C<-and>).
   90: 
   91: =item *
   92: 
   93: B<Automatic joins.>  Add a C<join> parameter to any select method to
   94: combine tables with INNER, LEFT, RIGHT, FULL, or CROSS joins.
   95: 
   96: =item *
   97: 
   98: B<Chained query builder.>  The C<query()> method returns a
   99: L<Database::Abstraction::Query> object for fluent, composable queries:
  100: C<< $db->query->where(â€¦)->order_by(â€¦)->limit(â€¦)->all() >>.
  101: 
  102: =item *
  103: 
  104: B<Schema introspection.>  C<columns()> lists column names; C<schema()>
  105: returns full type/nullability metadata, using native driver introspection
  106: (C<PRAGMA table_info> for SQLite, C<column_info> for others).
  107: 
  108: =item *
  109: 
  110: B<DSN portability.>  Pass a C<dsn> (plus optional C<username>/C<password>)
  111: to connect to any DBI-supported database (SQLite, PostgreSQL, MySQL, â€¦)
  112: instead of pointing at a local file.
  113: 
  114: =item *
  115: 
  116: B<Performance.>  Small files are slurped into a RAM hash for sub-millisecond
  117: lookups.  All DBI statement handles are cached with C<prepare_cached()>.
  118: A CHI-compatible cache layer is also supported.
  119: 
  120: =back

 2409: 
 2410: The table below lists every error that the module can croak or carp, what
 2411: triggers it, and how to resolve it.
 2412: 
 2413: =over 4
 2414: 
 2415: =item C<< I<Class>: abstract class >>
 2416: 
 2417: Direct instantiation of C<Database::Abstraction> was attempted.
 2418: Create a subclass and instantiate that instead.
 2419: 
 2420: =item C<< I<Class>: where are the files? >>
 2421: 
 2422: Neither C<directory> nor C<dsn> was supplied to C<new()>.
 2423: 
 2424: =item C<< I<Class>: I</path> is not a directory >>
 2425: 
 2426: The C<directory> argument exists on disk but is not a directory.
 2427: 
 2428: =item C<< I<Class>: cannot connect: I<$DBI::errstr> >>
 2429: 
 2430: DBI failed to connect to the given C<dsn>.  Check credentials and host.
 2431: 
 2432: =item C<< Can't find a file called 'I<name>' for the table I<T> in I<dir> >>
 2433: 
 2434: None of the probe extensions (C<.sql>, C<.psv>, C<.csv>, C<.db>, C<.xml>)
 2435: matched in C<directory>.
 2436: 
 2437: =item C<< I<Class>: prepare failed: I<$errstr> >>
 2438: 
 2439: C<prepare_cached()> returned false.  Usually a syntax error in an internally
 2440: built query; file a bug if you see this from a normal API call.
 2441: 
 2442: =item C<< _build_where_conditions: unsafe column name 'I<name>' >>
 2443: 
 2444: A criteria key contained characters outside C<[A-Za-z0-9_.]>.
 2445: This is a SQL-injection guard.  Use only valid SQL identifier characters.
 2446: 
 2447: =item C<< join: missing "table" >> / C<< join: missing "on" condition >>
 2448: 
 2449: A join spec hashref is incomplete.  Both C<table> and C<on> are required.
 2450: 
 2451: =item C<< Invalid JOIN type: I<TYPE> >>
 2452:

 2454: 
 2455: =item C<< I<Class>: Unknown column I<col> >> / C<< I<Class>: AUTOLOAD disabled >>
 2456: 
 2457: An AUTOLOAD call was made for a column that does not exist, or AUTOLOAD
 2458: was disabled with C<< auto_load => 0 >>.
 2459: 
 2460: =item C<< Usage: set_logger(logger => $logger) >>
 2461: 
 2462: C<set_logger()> was called without a C<logger> argument.
 2463:

 2465:

 2469:

 2472:

 2481: =head1 KNOWN LIMITATIONS
 2482: 
 2483: =over 4
 2484: 
 2485: =item *
 2486: 
 2487: B<Read-only.>  No INSERT, UPDATE, or DELETE is provided.  C<execute()>
 2488: runs raw read-only SQL.
 2489: 
 2490: =item *
 2491: 
 2492: B<Default CSV separator is C<!>>, not C<,>, for historical reasons.
 2493: Pass C<< sep_char => ',' >> for standard RFC 4180 files.
 2494: 
 2495: =item *
 2496: 
 2497: B<Primary-key column is named C<entry>>, not C<key>, because C<key>
 2498: is a SQL reserved word.  Override with the C<id> parameter.
 2499: 
 2500: =item *
 2501: 
 2502: B<XML slurp is limited.>  Only simple flat XML structures are supported
 2503: in slurp mode.  Multi-key or deeply nested documents will croak.
 2504: Force SQL mode with C<< max_slurp_size => 0 >> if slurp fails.
 2505: 
 2506: =item *
 2507: 
 2508: B<Unique key assumption in slurp mode.>  Duplicate values in the key
 2509: column silently overwrite earlier rows.  Disable slurp with
 2510: C<< max_slurp_size => 0 >> if duplicates are expected.
 2511: 
 2512: =item *
 2513: 
 2514: B<BerkeleyDB does not support joins or the chained query builder.>
 2515: 
 2516: =item *
 2517: 
 2518: B<Column names must be valid SQL identifiers> (letters, digits,
 2519: underscores, and a single dot for C<table.column> join notation).
 2520: Other characters will cause a croak.
 2521: 
 2522: =item *
 2523: 
 2524: B<count() cache is opportunistic.>  Count results are served from cache
 2525: only when a prior C<selectall_arrayref()> or C<count()> call with the
 2526: same criteria has already populated it.
 2527: 
 2528: =back
