   69: =head1 DESCRIPTION
   70: 
   71: C<Database::Abstraction> is a read-only ORM for Perl that gives a uniform
   72: interface over CSV, PSV, XML, SQLite, and BerkeleyDB files â€” without writing
   73: any SQL.
   74: 
   75: Key features:
   76: 
   77: =over 4
   78: 
   79: =item *
   80: 
   81: B<No SQL required.>  Use plain Perl method calls for simple lookups and
   82: scans; switch storage formats without changing application code.
   83: 
   84: =item *
   85: 
   86: B<Rich query criteria.>  Pass plain values, SQL wildcards, C<undef> (IS NULL),
   87: comparison operators (C<< > >> C<< < >> C<< >= >> C<< <= >> C<!=>), pattern
   88: operators (C<-like>, C<-not_like>), set operators (C<-in>, C<-not_in>,
   89: C<-between>), and logical groupings (C<-or>, C<-and>).
   90: 
   91: =item *
   92: 
   93: B<Automatic joins.>  Add a C<join> parameter to any select method to
   94: combine tables with INNER, LEFT, RIGHT, FULL, or CROSS joins.
   95: 
   96: =item *
   97: 
   98: B<Chained query builder.>  The C<query()> method returns a
   99: L<Database::Abstraction::Query> object for fluent, composable queries:
  100: C<< $db->query->where(â€¦)->order_by(â€¦)->limit(â€¦)->all() >>.
  101: 
  102: =item *
  103: 
  104: B<Schema introspection.>  C<columns()> lists column names; C<schema()>
  105: returns full type/nullability metadata, using native driver introspection
  106: (C<PRAGMA table_info> for SQLite, C<column_info> for others).
  107: 
  108: =item *
  109: 
  110: B<DSN portability.>  Pass a C<dsn> (plus optional C<username>/C<password>)
  111: to connect to any DBI-supported database (SQLite, PostgreSQL, MySQL, â€¦)
  112: instead of pointing at a local file.
  113: 
  114: =item *
  115: 
  116: B<Performance.>  Small files are slurped into a RAM hash for sub-millisecond
  117: lookups.  All DBI statement handles are cached with C<prepare_cached()>.
  118: A CHI-compatible cache layer is also supported.
  119: 
  120: =back

 2434: A criteria key contained characters outside C<[A-Za-z0-9_.]>.
 2435: This is a SQL-injection guard.  Use only valid SQL identifier characters.
 2436: 
 2437: =item C<< join: missing "table" >> / C<< join: missing "on" condition >>
 2438: 
 2439: A join spec hashref is incomplete.  Both C<table> and C<on> are required.
 2440: 
 2441: =item C<< Invalid JOIN type: I<TYPE> >>
 2442: 
 2443: C<type> in a join spec was not one of C<INNER LEFT RIGHT FULL CROSS>.
 2444: 
 2445: =item C<< I<Class>: Unknown column I<col> >> / C<< I<Class>: AUTOLOAD disabled >>
 2446: 
 2447: An AUTOLOAD call was made for a column that does not exist, or AUTOLOAD
 2448: was disabled with C<< auto_load => 0 >>.
 2449: 
 2450: =item C<< Usage: set_logger(logger => $logger) >>
 2451: 
 2452: C<set_logger()> was called without a C<logger> argument.
 2453: 
 2454: =item C<< Usage: execute(query => $query) >>
 2455: 
 2456: C<execute()> was called without a C<query> argument.
 2457: 
 2458: =item C<< XML slurp: I<...> is not yet supported >>
 2459: 
 2460: The XML file structure is too complex for slurp mode.
 2461: Use C<< max_slurp_size => 0 >> to force the DBI/XMLSimple SQL path.
 2462: 
 2463: =item C<< I<Class>: I<method> is meaningless on a NoSQL database >>
 2464: 
 2465: A relational method (C<selectall_arrayref>, C<count>, C<execute>, etc.)
 2466: was called on a BerkeleyDB backend, which only supports key-value lookup
 2467: via C<fetchrow_hashref>.

 2471: =head1 KNOWN LIMITATIONS
 2472: 
 2473: =over 4
 2474: 
 2475: =item *
 2476: 
 2477: B<Read-only.>  No INSERT, UPDATE, or DELETE is provided.  C<execute()>
 2478: runs raw read-only SQL.
 2479: 
 2480: =item *
 2481: 
 2482: B<Default CSV separator is C<!>>, not C<,>, for historical reasons.
 2483: Pass C<< sep_char => ',' >> for standard RFC 4180 files.
 2484: 
 2485: =item *
 2486: 
 2487: B<Primary-key column is named C<entry>>, not C<key>, because C<key>
 2488: is a SQL reserved word.  Override with the C<id> parameter.
 2489: 
 2490: =item *
 2491: 
 2492: B<XML slurp is limited.>  Only simple flat XML structures are supported
 2493: in slurp mode.  Multi-key or deeply nested documents will croak.
 2494: Force SQL mode with C<< max_slurp_size => 0 >> if slurp fails.
 2495: 
 2496: =item *
 2497: 
 2498: B<Unique key assumption in slurp mode.>  Duplicate values in the key
 2499: column silently overwrite earlier rows.  Disable slurp with
 2500: C<< max_slurp_size => 0 >> if duplicates are expected.
 2501: 
 2502: =item *
 2503: 
 2504: B<BerkeleyDB does not support joins or the chained query builder.>
 2505: 
 2506: =item *
 2507: 
 2508: B<Column names must be valid SQL identifiers> (letters, digits,
 2509: underscores, and a single dot for C<table.column> join notation).
 2510: Other characters will cause a croak.
 2511: 
 2512: =item *
 2513: 
 2514: B<count() cache is opportunistic.>  Count results are served from cache
 2515: only when a prior C<selectall_arrayref()> or C<count()> call with the
 2516: same criteria has already populated it.
 2517: 
 2518: =back
