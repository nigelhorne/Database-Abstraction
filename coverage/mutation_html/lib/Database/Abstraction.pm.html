   69: =head1 DESCRIPTION
   70: 
   71: C<Database::Abstraction> is a read-only ORM for Perl that gives a uniform
   72: interface over CSV, PSV, XML, SQLite, and BerkeleyDB files â€” without writing
   73: any SQL.
   74: 
   75: Key features:
   76: 
   77: =over 4
   78: 
   79: =item *
   80: 
   81: B<No SQL required.>  Use plain Perl method calls for simple lookups and
   82: scans; switch storage formats without changing application code.
   83: 
   84: =item *
   85: 
   86: B<Rich query criteria.>  Pass plain values, SQL wildcards, C<undef> (IS NULL),
   87: comparison operators (C<< > >> C<< < >> C<< >= >> C<< <= >> C<!=>), pattern
   88: operators (C<-like>, C<-not_like>), set operators (C<-in>, C<-not_in>,
   89: C<-between>), and logical groupings (C<-or>, C<-and>).
   90: 
   91: =item *
   92: 
   93: B<Automatic joins.>  Add a C<join> parameter to any select method to
   94: combine tables with INNER, LEFT, RIGHT, FULL, or CROSS joins.
   95: 
   96: =item *
   97: 
   98: B<Chained query builder.>  The C<query()> method returns a
   99: L<Database::Abstraction::Query> object for fluent, composable queries:
  100: C<< $db->query->where(â€¦)->order_by(â€¦)->limit(â€¦)->all() >>.
  101: 
  102: =item *
  103: 
  104: B<Schema introspection.>  C<columns()> lists column names; C<schema()>
  105: returns full type/nullability metadata, using native driver introspection
  106: (C<PRAGMA table_info> for SQLite, C<column_info> for others).
  107: 
  108: =item *
  109: 
  110: B<DSN portability.>  Pass a C<dsn> (plus optional C<username>/C<password>)
  111: to connect to any DBI-supported database (SQLite, PostgreSQL, MySQL, â€¦)
  112: instead of pointing at a local file.
  113: 
  114: =item *
  115: 
  116: B<Performance.>  Small files are slurped into a RAM hash for sub-millisecond
  117: lookups.  All DBI statement handles are cached with C<prepare_cached()>.
  118: A CHI-compatible cache layer is also supported.
  119: 
  120: =back

 2300: 
 2301: The table below lists every error that the module can croak or carp, what
 2302: triggers it, and how to resolve it.
 2303: 
 2304: =over 4
 2305: 
 2306: =item C<< I<Class>: abstract class >>
 2307: 
 2308: Direct instantiation of C<Database::Abstraction> was attempted.
 2309: Create a subclass and instantiate that instead.
 2310: 
 2311: =item C<< I<Class>: where are the files? >>
 2312: 
 2313: Neither C<directory> nor C<dsn> was supplied to C<new()>.
 2314: 
 2315: =item C<< I<Class>: I</path> is not a directory >>
 2316: 
 2317: The C<directory> argument exists on disk but is not a directory.
 2318: 
 2319: =item C<< I<Class>: cannot connect: I<$DBI::errstr> >>
 2320: 
 2321: DBI failed to connect to the given C<dsn>.  Check credentials and host.
 2322: 
 2323: =item C<< Can't find a file called 'I<name>' for the table I<T> in I<dir> >>
 2324: 
 2325: None of the probe extensions (C<.sql>, C<.psv>, C<.csv>, C<.db>, C<.xml>)
 2326: matched in C<directory>.
 2327: 
 2328: =item C<< I<Class>: prepare failed: I<$errstr> >>
 2329: 
 2330: C<prepare_cached()> returned false.  Usually a syntax error in an internally
 2331: built query; file a bug if you see this from a normal API call.
 2332: 
 2333: =item C<< _build_where_conditions: unsafe column name 'I<name>' >>
 2334: 
 2335: A criteria key contained characters outside C<[A-Za-z0-9_.]>.
 2336: This is a SQL-injection guard.  Use only valid SQL identifier characters.
 2337: 
 2338: =item C<< join: missing "table" >> / C<< join: missing "on" condition >>
 2339: 
 2340: A join spec hashref is incomplete.  Both C<table> and C<on> are required.
 2341: 
 2342: =item C<< Invalid JOIN type: I<TYPE> >>
 2343: 
 2344: C<type> in a join spec was not one of C<INNER LEFT RIGHT FULL CROSS>.
 2345: 
 2346: =item C<< I<Class>: Unknown column I<col> >> / C<< I<Class>: AUTOLOAD disabled >>
 2347: 
 2348: An AUTOLOAD call was made for a column that does not exist, or AUTOLOAD
 2349: was disabled with C<< auto_load => 0 >>.
 2350: 
 2351: =item C<< Usage: set_logger(logger => $logger) >>
 2352: 
 2353: C<set_logger()> was called without a C<logger> argument.
 2354: 
 2355: =item C<< Usage: execute(query => $query) >>
 2356: 
 2357: C<execute()> was called without a C<query> argument.
 2358: 
 2359: =item C<< XML slurp: I<...> is not yet supported >>
 2360: 
 2361: The XML file structure is too complex for slurp mode.
 2362: Use C<< max_slurp_size => 0 >> to force the DBI/XMLSimple SQL path.
 2363: 
 2364: =item C<< I<Class>: I<method> is meaningless on a NoSQL database >>
 2365: 
 2366: A relational method (C<selectall_arrayref>, C<count>, C<execute>, etc.)
 2367: was called on a BerkeleyDB backend, which only supports key-value lookup
 2368: via C<fetchrow_hashref>.

 2370: =back

 2372: =head1 KNOWN LIMITATIONS
 2373: 
 2374: =over 4
 2375: 
 2376: =item *
 2377: 
 2378: B<Read-only.>  No INSERT, UPDATE, or DELETE is provided.  C<execute()>
 2379: runs raw read-only SQL.
 2380: 
 2381: =item *
 2382: 
 2383: B<Default CSV separator is C<!>>, not C<,>, for historical reasons.
 2384: Pass C<< sep_char => ',' >> for standard RFC 4180 files.
 2385: 
 2386: =item *
 2387: 
 2388: B<Primary-key column is named C<entry>>, not C<key>, because C<key>
 2389: is a SQL reserved word.  Override with the C<id> parameter.
 2390: 
 2391: =item *
 2392: 
 2393: B<XML slurp is limited.>  Only simple flat XML structures are supported
 2394: in slurp mode.  Multi-key or deeply nested documents will croak.
 2395: Force SQL mode with C<< max_slurp_size => 0 >> if slurp fails.
 2396: 
 2397: =item *
 2398: 
 2399: B<Unique key assumption in slurp mode.>  Duplicate values in the key
 2400: column silently overwrite earlier rows.  Disable slurp with
 2401: C<< max_slurp_size => 0 >> if duplicates are expected.
 2402: 
 2403: =item *
 2404: 
 2405: B<BerkeleyDB does not support joins or the chained query builder.>
 2406: 
 2407: =item *
 2408: 
 2409: B<Column names must be valid SQL identifiers> (letters, digits,
 2410: underscores, and a single dot for C<table.column> join notation).
 2411: Other characters will cause a croak.
 2412: 
 2413: =item *
 2414: 
 2415: B<count() cache is opportunistic.>  Count results are served from cache
 2416: only when a prior C<selectall_arrayref()> or C<count()> call with the
 2417: same criteria has already populated it.
 2418: 
 2419: =back
