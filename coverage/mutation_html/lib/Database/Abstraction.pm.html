   68: =head1 DESCRIPTION
   69: 
   70: C<Database::Abstraction> is a read-only ORM for Perl that gives a uniform
   71: interface over CSV, PSV, XML, SQLite, and BerkeleyDB files â€” without writing
   72: any SQL.
   73: 
   74: Key features:
   75: 
   76: =over 4
   77: 
   78: =item *
   79: 
   80: B<No SQL required.>  Use plain Perl method calls for simple lookups and
   81: scans; switch storage formats without changing application code.
   82: 
   83: =item *
   84: 
   85: B<Rich query criteria.>  Pass plain values, SQL wildcards, C<undef> (IS NULL),
   86: comparison operators (C<< > >> C<< < >> C<< >= >> C<< <= >> C<!=>), pattern
   87: operators (C<-like>, C<-not_like>), set operators (C<-in>, C<-not_in>,
   88: C<-between>), and logical groupings (C<-or>, C<-and>).
   89: 
   90: =item *
   91: 
   92: B<Automatic joins.>  Add a C<join> parameter to any select method to
   93: combine tables with INNER, LEFT, RIGHT, FULL, or CROSS joins.
   94: 
   95: =item *
   96: 
   97: B<Chained query builder.>  The C<query()> method returns a
   98: L<Database::Abstraction::Query> object for fluent, composable queries:
   99: C<< $db->query->where(â€¦)->order_by(â€¦)->limit(â€¦)->all() >>.
  100: 
  101: =item *
  102: 
  103: B<Schema introspection.>  C<columns()> lists column names; C<schema()>
  104: returns full type/nullability metadata, using native driver introspection
  105: (C<PRAGMA table_info> for SQLite, C<column_info> for others).
  106: 
  107: =item *
  108: 
  109: B<DSN portability.>  Pass a C<dsn> (plus optional C<username>/C<password>)
  110: to connect to any DBI-supported database (SQLite, PostgreSQL, MySQL, â€¦)
  111: instead of pointing at a local file.
  112: 
  113: =item *
  114: 
  115: B<Performance.>  Small files are slurped into a RAM hash for sub-millisecond
  116: lookups.  All DBI statement handles are cached with C<prepare_cached()>.
  117: A CHI-compatible cache layer is also supported.
  118: 
  119: =back

 2496: =head1 KNOWN LIMITATIONS
 2497: 
 2498: =over 4
 2499: 
 2500: =item *
 2501: 
 2502: B<Read-only.>  No INSERT, UPDATE, or DELETE is provided.  C<execute()>
 2503: runs raw read-only SQL.
 2504: 
 2505: =item *
 2506: 
 2507: B<Default CSV separator is C<!>>, not C<,>, for historical reasons.
 2508: Pass C<< sep_char => ',' >> for standard RFC 4180 files.
 2509: 
 2510: =item *
 2511: 
 2512: B<Primary-key column is named C<entry>>, not C<key>, because C<key>
 2513: is a SQL reserved word.  Override with the C<id> parameter.
 2514: 
 2515: =item *
 2516: 
 2517: B<XML slurp is limited.>  Only simple flat XML structures are supported
 2518: in slurp mode.  Multi-key or deeply nested documents will croak.
 2519: Force SQL mode with C<< max_slurp_size => 0 >> if slurp fails.
 2520: 
 2521: =item *
 2522: 
 2523: B<Unique key assumption in slurp mode.>  Duplicate values in the key
 2524: column silently overwrite earlier rows.  Disable slurp with
 2525: C<< max_slurp_size => 0 >> if duplicates are expected.
 2526: 
 2527: =item *
 2528: 
 2529: B<BerkeleyDB does not support joins or the chained query builder.>
 2530: 
 2531: =item *
 2532: 
 2533: B<Column names must be valid SQL identifiers> (letters, digits,
 2534: underscores, and a single dot for C<table.column> join notation).
 2535: Other characters will cause a croak.
 2536: 
 2537: =item *
 2538: 
 2539: B<count() cache is opportunistic.>  Count results are served from cache
 2540: only when a prior C<selectall_arrayref()> or C<count()> call with the
 2541: same criteria has already populated it.

 2543: =back
