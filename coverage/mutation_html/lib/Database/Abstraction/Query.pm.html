   65: =head1 DESCRIPTION
   66: 
   67: C<Database::Abstraction::Query> is a fluent query builder returned by
   68: C<< $db->query() >>.  You assemble a query by chaining builder methods,
   69: then execute it with a terminal method.
   70: 
   71: =over 4
   72: 
   73: =item *
   74: 
   75: B<Builder methods> (C<select>, C<where>, C<join>, C<order_by>, C<limit>,
   76: C<offset>) all return C<$self>, so calls can be chained in any order.
   77: 
   78: =item *
   79: 
   80: B<Terminal methods> (C<all>, C<first>, C<count>) assemble the SQL, execute
   81: it, and return the result.  Each terminal method can be called on the same
   82: builder object independently â€” calling C<first()> does not modify the stored
   83: state (it temporarily sets LIMIT 1 internally).
   84: 
   85: =item *
   86: 
   87: C<where()> calls are B<merged with AND semantics>: each call adds more
   88: required conditions.  To express OR conditions pass C<< -or => [...] >> as a
   89: key inside a single C<where()> call.
   90: 
   91: =item *
   92: 
   93: The C<join> parameter accepts the same spec as the C<join> parameter in
   94: L<Database::Abstraction/QUERY CRITERIA>.
   95: 
   96: =item *
   97: 
   98: BerkeleyDB backends are not supported.
   99: 
  100: =back
